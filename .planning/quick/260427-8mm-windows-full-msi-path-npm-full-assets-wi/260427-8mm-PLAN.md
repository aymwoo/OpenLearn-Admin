# Quick Task 260427-8mm Plan

## Goal

把 Windows online/full 两条 Node.js MSI 安装链路再梳理并固化一遍：online
版继续负责下载 MSI 和进度回传，full 版继续依赖打包进

## Threat model

### Trust boundaries

| Boundary | Description |
|----------|-------------|
| Internet download → local MSI execution | online 版会下载并执行 MSI，必须把下载阶段与安装后 readiness 失败区分开 |
| GitHub Actions bundle resources → runtime resource scan | full 版依赖 CI 把 MSI 放到运行时扫描目录，路径漂移会直接导致 bundled 安装失效 |
| Installed Node directory → in-app command execution | MSI 安装成功不等于当前应用进程内 npm 可用，PATH 注入或显式命令解析出错就会失败 |

### STRIDE threat register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-quick-01 | T | Windows full bundle contract | mitigate | workflow 固定 `assets-windows -> resources/nodejs`，运行时测试与文案都只围绕该目录契约 |
| T-quick-02 | D | `install_node_env` Windows branching | mitigate | 把 bundled/online 分支与共同 readiness 收尾显式收口，避免 silent fallback 或分支漂移 |
| T-quick-03 | D | in-app npm execution after install | mitigate | 保持 `ensure_windows_node_ready` 与 Windows 命令 helper 为唯一成功门槛，并用单元测试锁住路径组装契约 |

## Success criteria

- Windows online/full 安装逻辑边界清晰，后续维护时不需要再猜 full 是否会回退 online。
- full 版打包资源目录与运行时扫描目录保持同一契约。
- 程序内 npm/pnpm 使用链路继续绑定刚安装的 Windows Node 目录。
- 至少有一组 Rust 单元测试专门防止 Windows MSI 扫描与命令路径逻辑回归。
