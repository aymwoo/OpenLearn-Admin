## 2024-05-18 - Caching sysinfo's Disks in Tauri backend
**Learning:** Instantiating `Disks::new_with_refreshed_list()` in `get_system_info` is an expensive operation that runs continuously when polling system metrics. Furthermore, since sysinfo v0.36.1 doesn't have `refresh_list()` directly on the instance, we should cache `Disks` and call `.refresh(true)` to update it, avoiding full reallocation and disk re-discovery overhead. Selectively refreshing metrics using `sys.refresh_cpu_usage()` and `sys.refresh_memory()` instead of `sys.refresh_all()` also reduces CPU usage overhead significantly.
**Action:** Always cache long-lived struct like `sysinfo::Disks` in `AppState` when periodically polling system states to minimize expensive object re-creation. Only use selective `refresh_*` calls instead of `refresh_all()`.
