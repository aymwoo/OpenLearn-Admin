## 2024-04-28 - ⚡ Bolt: Optimize system info polling in Rust backend
**Learning:** The `sysinfo` crate in `src-tauri` was performing a full `sys.refresh_all()` and instantiating `Disks::new_with_refreshed_list()` on every call to `get_system_info`. This is a significant overhead when periodically polling system states, as it refreshes networks, components, users, processes, and disks from scratch.
**Action:** For optimal performance when periodically polling system states, avoid calling `sys.refresh_all()`. Instead, selectively refresh only the required metrics (e.g., `sys.refresh_cpu_usage()`, `sys.refresh_memory()`). Additionally, cache `Disks` in `AppState` and update it using `disks.refresh(true)` instead of repeatedly instantiating `Disks::new_with_refreshed_list()`. Note: For sysinfo v0.36.1, `refresh_list()` method doesn't exist, use `refresh(true)` to update disk lists and data.
